run

PURPOSE

gui_test_runner/run is an alternative method to execute the graphical user interface of mlUnit.

SYNOPSIS

function self = run(self, test, dock, shorten)

DESCRIPTION 

gui_test_runner/run is an alternative method to execute the graphical 
user interface of mlUnit. 

  Example
  =======
  With the second parameter it is possible to specify a test method, test
  case or test suite, that is executed immediately after the start of the
  graphical user interface. 
         Example: run(gui_test_runner, 'mlunit_all_tests');

  See also GUI_TEST_RESULT, GUI_TEST_RUNNER.

CROSS-REFERENCE INFORMATION

This function calls:
  • get_handle gui_test_runner/get_handle returns the graphics handle of the gui window, if the singleton object is passed as the input parameter.
  • get_object gui_test_runner/get_object returns the singleton object of the gui window.
  • gui gui_test_runner/gui execute the graphical user interface of mlUnit.
This function is called by:
  • gui gui_test_runner/gui execute the graphical user interface of mlUnit.
  • load gui_test_runner/loads starts the graphical user interface of mlUnit with saved parameters from the .mat-file mlunit.tmp.
  • run mlunit/run executes the the graphical user interface of mlUnit.
  • run test_suite/run executes the test suite and saves the results in result.
  • run text_test_runner/run execute the test and writes the results to a stream in textual form (using TextTestResult).
