filter_lasterror_wraps

PURPOSE

Free error message of leading 'Error:' or 'Error using' lines.

SYNOPSIS

function [message, stack] = filter_lasterror_wraps(self)

DESCRIPTION 

Free error message of leading 'Error:' or 'Error using' lines.
  Those are MATLAB byproducts of using lasterror. lasterror adds a line to the
  front of the error message. For runtime errors, this is 'Error using',
  followed by the function name where the error occurred. For syntax errors,
  this is 'Error:' followed by the function and line where the error occurred.
  Strangely, for syntax errors these extra information are not included in the
  stack. filter_lasterror_wraps therefore parses the extra line, and adds the
  information as stack item.
  filter_lasterror_wraps works on the internally buffered error information
  that you provided in mlunit_errorinfo's constructor.

  [message, stack] = filter_lasterror_wraps(self) returns the cropped message,
  and the stack structure, that may be enhanced in case of a syntax error.

  This method is expected to be private. Rather use get_message_with_stack
  instead.

  See also test_filter_lasterror, get_message_with_stack

0027 % use verbatim values as defaults
0028 message = self.err.message;
0029 stack = self.err.stack;

0031 % lasterror wraps runtime errors with either:
0032 %    'Error using ==>...' on R2006b to R2010b, or
0033 %    'Error using <a href...' from R2011b on
0034 regexp_runtime_err = ['^Error using ' ...   % always starts with 'Error using'
0035                       '(==>|<a href)' ...   % '==>' before R2011b, as <a> after
0036                       '[^\n]*' ...          % skip anything on this line
0037                       '\n(.*)'];            % capture the next line(s)
0038 num_captures_runtime_err = 2;               % 2 capture groups if successful
0039                   
0040 % lasterror wraps syntax errors really awkwardly across the releases
0041 % for details, see test_mlunit_errorinfo
0042 regexp_syntax_err = ['^Error: ' ...         % starts with 'Error:'
0043                      '(<a[^>]*>)?' ...      % puts an anchor around the file
0044                      'File: ' ...
0045                      '([\w\ \.,$&\/\(\)\\:@]+.[mp])' ...  % file name or path
0046                      ' Line: (\d+)' ...
0047                      ' Column: (\d+)' ...
0048                      '.*' ...               % any further character
0049                      '(<\/a>\n|\n<\/a>|\n)' ... % anchor closing tag may be before or after newline or none at all
0050                      '(.*)'];               % this is the actual message
0051 num_captures_syntax_err = 6;                % 6 capture groups if successful
0052 
0053 % run both
0054 tokens_runtime = regexp(message, regexp_runtime_err, 'tokens', 'once');
0055 tokens_syntax = regexp(message, regexp_syntax_err, 'tokens', 'once');
0056 
0057 % match either a runtime or a syntax error
0058 if length(tokens_runtime) == num_captures_runtime_err
0059     message = char(tokens_runtime(2));
0060 elseif length(tokens_syntax) == num_captures_syntax_err
0061     message = char(tokens_syntax(6));
0062     file = char(tokens_syntax(2));
0063     line = str2double(char(tokens_syntax(3)));
0064     
0065     % see if we can resolve the full file path

0067     if isempty(fullname), fullname = file; end
0068     
0069     % push reconstructed call location onto stack
0070     stackitem = struct('file', {fullname}, 'line', {line}, 'name', {file});

0072 else
0073     % unrecognized message format: leave message and stack as they are
0074 end
